# Polars Data System Migration Design

## Scope

Migrate active Python code and pytest coverage in this repository from pandas-first data handling to Polars-first data handling. Preserve current file layout and command entry points. Include tick-data-safe infrastructure so `input-data/merged_nq_ticks.parquet` is accessed through lazy scans or streaming collects, not eager full-file loads.

Excluded from migration: prose-only historical docs under `docs/plans/`; they describe past work and can keep pandas snippets. Generated outputs stay unchanged except when regenerated by scripts.

## Architecture

Use Polars as the project dataframe API. Core IO lives in `utils/minute_bars.py` for minute bars and a new small tick module for tick parquet metadata/lazy scans. Scripts call these helpers instead of directly reading all data with pandas. Plotting scripts may convert final small result tables to pandas/numpy only at matplotlib boundaries if needed, but data prep stays Polars.

Tick data flow: `scan_tick_data(path)` returns `pl.LazyFrame`; `get_tick_schema(path)` reads metadata only; downstream experiments filter by date/session first, then `collect(streaming=True)` only for bounded windows.

## Components

- `utils/minute_bars.py`: Polars read/normalize/session-window helpers.
- `utils/tick_data.py`: schema constants, metadata read, lazy scan, bounded-window collectors, optional tick-to-minute aggregation.
- Entry scripts: `session_tagger.py`, `macro_outcomes.py`, feature scripts, viz scripts migrate to Polars operations.
- Tests: replace pandas fixtures/assertions with Polars, plus tests proving tick helpers use lazy scans and metadata access.
- Dependency metadata: add a minimal `requirements.txt` so Polars is explicit.

## Error handling

Validate required columns early with clear `ValueError`. Never expose helper names that imply eager loading for full tick parquet. For tick bounded collect, require at least one time predicate/window argument.

## Testing

Use TDD for each migrated module: first tests assert Polars return types and existing outputs. Then convert implementation. Baseline: `python -m pytest features/lrlr/test features/trend -q` already passes in worktree. Final: run full relevant suite: `python -m pytest test features/lrlr/test features/trend -q`, plus script smoke tests where input fixtures exist.
