# Macro FVG Volume Delta Dominance Design

## Goal

Add a clean extension to the existing macro FVG study that tests whether 5-second tick volume-delta dominance around a macro FVG is associated with the FVG's expected win rate.

The study should compare current macro FVG win metrics across quantiles of:

1. **Aligned aggressor dominance**: delta in the same direction as the FVG side.
2. **Absolute aggressor dominance**: magnitude of one-sided aggression regardless of side.

This should be integrated with the current macro FVG report rather than becoming a detached analysis.

## Existing Inputs

### Macro FVG events

Path: `outputs/nq_macro_fvg_events.parquet`

Generated by `features/macro_fvg_study.py`. Relevant columns include:

- `date`
- `fvg_side`
- `assigned_at`
- `confirmed_at`
- `is_confirmable_by_1559`
- `retraced_by_1559`
- `successful_by_1559`
- `mfe_pct_to_1559`
- `mae_pct_to_1559`

The existing win condition remains authoritative. The primary win-rate metric is `successful_share_of_confirmable`, computed as successful FVGs divided by confirmable FVGs.

### 5-second macro volume delta

Path: `outputs/nq_macro_volume_delta_5s.parquet`

Generated by `volume_delta.py`. Relevant columns include:

- `datetime_utc`
- `trade_date_et`
- `macro_bucket_index`
- `buy_size`
- `sell_size`
- `classified_size`
- `total_size`
- `volume_delta`
- `delta_imbalance`
- `buy_ticks`
- `sell_ticks`
- `tick_delta`
- `classified_share`
- `is_empty`

`macro_bucket_index` covers 0 through 119 for the 15:50:00-15:59:59 ET macro window, with each bucket representing 5 seconds.

## Chosen Approach

Use a hybrid design:

- Add a small focused helper module for volume-delta enrichment and quantile assignment.
- Call that helper from `features/macro_fvg_study.py` so the existing FVG study remains the single integrated report entry point.
- Append new summary scopes to `outputs/nq_macro_fvg_summary.parquet`.

This keeps the analysis integrated with the previous FVG work while avoiding more bulk inside the already-large `macro_fvg_study.py`.

## Event-to-Delta Join

Each FVG should be joined to one 5-second volume-delta bucket using the FVG's `confirmed_at` timestamp.

Rationale:

- The FVG is only known after bar 3 completes.
- Using `confirmed_at` aligns the delta observation with the moment the setup becomes actionable in the existing study.
- This avoids lookahead beyond the confirmation moment.

Join logic:

1. Treat `confirmed_at` as ET market time.
2. Keep only events whose `confirmed_at` falls inside 15:50:00-15:59:59 ET.
3. Compute seconds since 15:50:00 ET.
4. Compute `macro_bucket_index = floor(seconds_since_macro_start / 5)`.
5. Join on:
   - event `date` to delta `trade_date_et`
   - computed `macro_bucket_index` to delta `macro_bucket_index`

If the matching volume-delta bucket is absent or the delta value is null, the dominance metrics and quantile labels remain null and those rows are excluded from quantile grouped summaries.

## Dominance Features

Add these event-level columns after enrichment:

- `fvg_delta_bucket_index`
- `fvg_delta_volume_delta`
- `fvg_delta_imbalance`
- `fvg_delta_tick_delta`
- `fvg_delta_classified_share`
- `fvg_delta_total_size`
- `fvg_delta_is_empty`
- `aligned_delta_imbalance`
- `abs_delta_imbalance`
- `aligned_volume_delta`
- `abs_volume_delta`
- `aligned_tick_delta`
- `abs_tick_delta`

Definitions:

- For bullish FVGs, aligned values equal the raw delta values.
- For bearish FVGs, aligned values are the negative of raw delta values.
- Absolute values use `abs(...)` and ignore FVG side.

The primary quantile feature is `delta_imbalance`, because it normalizes by classified size and best represents aggressor dominance independent of total activity. Raw volume and tick delta columns are retained as diagnostics and for future extensions.

## Quantile Buckets

Create quartile labels for both primary dominance views:

- `aligned_delta_imbalance_quantile`
- `abs_delta_imbalance_quantile`

Labels:

- `q1_lowest`
- `q2`
- `q3`
- `q4_highest`

Rules:

- Default bucket count is 4.
- Bucket assignment is rank-based over non-null values to avoid fragile behavior when duplicate values occur.
- If there are fewer unique non-null values than 4, use the number of unique values and label the available ordered buckets consistently from low to high.
- Null dominance values remain null and are excluded from quantile summary scopes.

## Summary Scopes

Append these scopes to the existing macro FVG summary output:

1. `success_context_aligned_delta_imbalance_quantile`
   - grouped by `aligned_delta_imbalance_quantile`
2. `success_context_abs_delta_imbalance_quantile`
   - grouped by `abs_delta_imbalance_quantile`
3. `success_context_side_aligned_delta_imbalance_quantile`
   - grouped by `fvg_side` and `aligned_delta_imbalance_quantile`
4. `success_context_side_abs_delta_imbalance_quantile`
   - grouped by `fvg_side` and `abs_delta_imbalance_quantile`

Each scope should use the existing success-context statistics:

- `n_total`
- `n_confirmable`
- `n_retraced`
- `n_successful`
- `retrace_rate`
- `success_after_retrace_rate`
- `successful_share_of_confirmable`
- successful-event MFE/MAE summary columns where applicable

The primary comparison is `successful_share_of_confirmable` across q1 through q4.

## Output Schema Changes

Extend `EVENT_COLUMNS` with the event-level enrichment and quantile columns listed above.

Extend `SUMMARY_COLUMNS` with nullable grouping columns:

- `aligned_delta_imbalance_quantile`
- `abs_delta_imbalance_quantile`

Existing summary rows should continue to write successfully with null values in these new columns.

## Error Handling

- If the 5-second volume-delta parquet is missing, the FVG study should still run and produce the existing report, but emit a clear warning that volume-delta dominance summaries were skipped.
- If the volume-delta parquet exists but lacks required columns, raise a clear `ValueError` naming the missing columns.
- If there are no joinable FVG rows, append no dominance summary scopes and keep the existing report intact.
- Do not eager-read the raw tick file for this study. The study consumes the already-built 5-second volume-delta parquet.

## Testing

Add focused pytest coverage with small synthetic Polars frames and temporary parquet files:

1. Enrichment joins FVG events to the correct `macro_bucket_index` from `confirmed_at`.
2. Bullish and bearish aligned dominance signs are correct.
3. Absolute dominance ignores FVG side.
4. Quantile assignment produces ordered low-to-high labels.
5. Dominance summary scopes use the existing success win condition and compute `successful_share_of_confirmable` correctly.
6. Missing delta parquet skips enrichment without breaking existing FVG summary generation.
7. Missing required delta columns raises `ValueError`.

Tests should not read `input-data/merged_nq_ticks.parquet`.

## Out of Scope

- Rebuilding volume-delta from ticks inside the FVG study.
- New charting in the first implementation pass.
- Modeling or statistical significance testing beyond descriptive quantile win-rate tables.
- Changing the existing FVG win condition.
- Replacing existing FVG summary scopes.
